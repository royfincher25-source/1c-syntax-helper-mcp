## Архитектурные решения

### 1. Хранение сессий

```python
# В startup():
app.state.sse_sessions = {}  # Инициализация при запуске

# В SSE endpoint:
request.app.state.sse_sessions[session_id] = message_queue

# В WebSocket endpoint:
app.state.sse_sessions[session_id] = message_queue  # Глобальный app
```

### 2. Ограничение очереди

```python
# Константа
SSE_QUEUE_MAX_SIZE = 100  # Максимум 100 сообщений

# Создание очереди
message_queue = asyncio.Queue(maxsize=SSE_QUEUE_MAX_SIZE)

# Обработка переполнения
try:
    queue.put_nowait(response_data)
except asyncio.QueueFull:
    return JSONResponse(status_code=503, content={"error": "Queue full"})
```

### 3. Таймаут сессии

```python
# Константа
SSE_SESSION_TIMEOUT_SECONDS = 3600  # 1 час

# Проверка в цикле
session_start = time.time()
while True:
    if time.time() - session_start > SSE_SESSION_TIMEOUT_SECONDS:
        logger.info(f"Session timeout for {session_id}")
        break
```

### 4. Очистка сессий

```python
# В finally блоке
try:
    del request.app.state.sse_sessions[session_id]
    logger.debug(f"SSE сессия удалена: {session_id}")
except KeyError:
    logger.debug(f"SSE сессия {session_id} уже удалена")
```
