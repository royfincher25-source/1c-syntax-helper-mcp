## 🔍 Технические детали

### Алгоритм работы:

```
1. Получение ответа от call_next
2. Проверка _should_compress():
   - status_code < 400
   - content-type не в exclude_list
3. Чтение тела ответа
4. Проверка размера >= min_size
5. Сжатие через gzip.compress()
6. Проверка эффективности сжатия
7. Возврат сжатого ответа с заголовком Content-Encoding: gzip
```

### Формат логов:

```json
{
  "timestamp": "2026-03-05T16:00:00.000Z",
  "level": "DEBUG",
  "message": "Gzip сжатие: POST /mcp",
  "original_size": 5120,
  "compressed_size": 1536,
  "compression_ratio_percent": 70.0
}
```
