# HBK Parser V2 - Результаты тестирования

## Дата: 2026-03-06

## Резюме

Разработан и протестирован новый парсер HBK файлов с использованием streaming подхода. Тестирование показало что **пакетное извлечение в temp директорию (текущий подход) эффективнее** чем streaming с отдельными вызовами 7zip.

## Анализ производительности

### Проблема

Streaming подход с отдельными вызовами `7z x -so` для каждого файла крайне медленный:
- Каждый вызов 7zip: ~0.5-0.7с накладных расходов
- 25511 файлов × 0.6с = **15306с (4.25 часа)** только на запуск процессов

### Сравнение подходов

| Подход | Время на файл | Общее время (25511 файлов) | Память | Диск |
|--------|---------------|---------------------------|--------|------|
| **V1: extract_all_to_temp** | ~0.01с* | ~10-15 мин | 256MB | 75MB temp |
| **V2: streaming (7z x -so)** | ~0.6с | ~4-5 часов | 100MB | 0MB |

*После пакетного извлечения

## Выводы

### Streaming подход (V2) - НЕ РЕКОМЕНДУЕТСЯ

**Преимущества:**
- Нет временных файлов
- Меньше памяти (100MB vs 256MB)

**Недостатки:**
- **Крайне низкая скорость** (1.5 файла/с vs ~28 файлов/с)
- 25000+ вызовов 7zip subprocess
- Прогноз полного парсинга: 4-5 часов

### Пакетное извлечение (V1) - РЕКОМЕНДУЕТСЯ

**Преимущества:**
- **Высокая скорость** после извлечения
- Один вызов 7zip для всех файлов
- Прогноз полного парсинга: 10-15 минут

**Недостатки:**
- Временные файлы на диске (~75MB)
- Больше памяти для кэша

## Рекомендации

### 1. Оптимизировать текущий парсер (V1)

Вместо замены на streaming подход, оптимизировать текущий:

```python
# Улучшения:
1. Кэширование результатов парсинга (не только файлов)
2. Параллельный парсинг HTML (asyncio.gather с semaphore)
3. Инкрементальная индексация (только новые файлы)
```

### 2. Улучшения для V1

**Файл: `src/parsers/hbk_parser.py`**

```python
# Добавить параллелизм:
semaphore = asyncio.Semaphore(10)  # Макс 10 параллельных задач

async def parse_batch(files):
    tasks = [parse_file(f, semaphore) for f in files]
    return await asyncio.gather(*tasks)
```

**Файл: `src/core/constants.py`**

```python
# Оптимизировать таймауты:
HBK_EXTRACT_TIMEOUT_BASE = 300  # 5 минут вместо 10
HBK_FILE_READ_TIMEOUT = 30      # 30с на файл
PARALLEL_PARSE_LIMIT = 10       # Параллельный парсинг
```

### 3. Альтернатива: py7zr библиотека

Рассмотреть использование `py7zr` (Python библиотека):

```bash
pip install py7zr
```

**Преимущества:**
- Нет зависимости от внешнего 7zip
- Прямое чтение в память
- Быстрее чем subprocess

**Недостатки:**
- Дополнительная зависимость
- Может не поддерживать поврежденные архивы
