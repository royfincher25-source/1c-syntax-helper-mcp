# Архитектура MCP Server

## Обзор

MCP Server - это сервер для предоставления справочной информации по 1С предприятию через протокол MCP (Model Context Protocol).

## Модули

### Core (`src/core/`)

| Модуль | Описание |
|--------|----------|
| `elasticsearch.py` | Клиент для Elasticsearch |
| `cache.py` | In-memory кэш с LRU/LFU стратегиями |
| `metrics.py` | Сбор метрик и мониторинг |
| `circuit_breaker.py` | Circuit breaker паттерн |
| `lifespan.py` | Управление жизненным циклом |
| `exception_handlers.py` | Обработчики исключений |
| `rate_limiter.py` | Rate limiting |

### Search (`src/search/`)

| Модуль | Описание |
|--------|----------|
| `search_service.py` | Фасад для поиска |
| `cache_service.py` | Логика кэширования поиска |
| `circuit_breaker_handler.py` | Fallback логика |
| `syntax_info_service.py` | Детальная справка |
| `context_search_service.py` | Контекстный поиск |
| `object_members_service.py` | Список членов объекта |
| `find_help_service.py` | Основной поиск |
| `examples_service.py` | Примеры кода |

### Handlers (`src/handlers/`)

| Модуль | Описание |
|--------|----------|
| `mcp_handler.py` | Обработчик MCP запросов |
| `mcp_formatter.py` | Форматтер ответов |
| `formatters/` | Специализированные форматтеры |

### Parsers (`src/parsers/`)

| Модуль | Описание |
|--------|----------|
| `hbk_parser.py` | Парсер .hbk файлов |
| `indexer.py` | Индексация в Elasticsearch |
| `sevenzip_manager.py` | Управление 7zip |

## Паттерны

### Facade
`main.py`, `search_service.py`, `mcp_formatter.py` - фасады, делегирующие работу специализированным модулям.

### Strategy
Стратегии вытеснения кэша: `LRUStrategy`, `LFUStrategy`.

### Circuit Breaker
Защита от каскадных отказов при проблемах с Elasticsearch.

### Dependency Injection
Использование `dependency_injection.py` для управления зависимостями.

## Производительность

- **Поиск**: < 100ms (p95) с кэшированием
- **Кэш**: LRU/LFU стратегии, настраиваемый размер
- **Индексация**: > 100 docs/sec с батч-обработкой

## Тестирование

- Unit тесты: `tests/core/`, `tests/parsers/`
- Integration тесты: `tests/integration/`
- Benchmarks: `tests/benchmarks/`

## Конфигурация

Основные настройки в `src/core/config.py`:
- `data.hbk_directory` - путь к .hbk файлам
- `elasticsearch.*` - настройки ES
- `cache.*` - настройки кэша
- `rate_limit.*` - лимиты запросов
